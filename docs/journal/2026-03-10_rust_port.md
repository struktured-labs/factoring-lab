# Rust Port of Digit Convolution Backtracking

**Date:** 2026-03-10

## Motivation

The digit convolution backtracking algorithm is compute-bound: it explores
a tree of digit assignments with carry-propagation constraints. Python's
interpreter overhead dominates on larger semiprimes. A compiled Rust
implementation should give a straightforward speedup without algorithmic
changes.

## What was done

1. **Created `rust/factoring_kernels/`** -- a PyO3/maturin crate that
   implements `digit_convolution_factor(n, base, max_digits)` in Rust,
   returning `(factor, cofactor, iterations)`.

2. **Python wrapper** at `src/factoring_lab/algorithms/digit_convolution_rs.py`
   exposes `DigitConvolutionRust`, which follows the `FactoringAlgorithm`
   interface and falls back gracefully when the extension is not built.

3. **Tests** in `tests/test_digit_convolution_rs.py` mirror the existing
   Python test suite and add parity checks confirming both implementations
   produce identical factors on the same inputs.

4. **Benchmark** (`scripts/rust_vs_python.py`) compares Python vs Rust on
   12, 16, 20, and 24-bit semiprimes.

## Benchmark results

| Bit size | Python avg (s) | Rust avg (s) | Speedup |
|----------|----------------|--------------|---------|
| 12       | 0.001162       | 0.000042     | 26x     |
| 16       | 0.010334       | 0.000330     | 31x     |
| 20       | 0.148054       | 0.004520     | 33x     |
| 24       | 0.473508       | 0.014088     | 33x     |

The Rust port achieves a consistent **~30-33x speedup** over Python.
Iteration counts are identical, confirming the two implementations follow
the exact same search path. The speedup is purely from eliminating
interpreter overhead (function call dispatch, dynamic typing, boxed
integers).

## Notes

- Used `u128` for the number type, supporting semiprimes up to ~38 decimal
  digits. For larger inputs, a big-integer crate would be needed.
- The speedup is remarkably stable across problem sizes, which is expected:
  the algorithm performs the same number of iterations, and each iteration
  does a fixed amount of arithmetic.
- Build: `cd rust/factoring_kernels && maturin develop --release`
